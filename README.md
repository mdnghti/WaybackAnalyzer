# Wayback Domain Analyzer

Утилита на Python для массового, автоматического анализа тематики доменов с использованием исторических снимков (snapshots) из сервиса Web Archive (Wayback Machine). 

Программа фильтрует домены по входным метрикам (Ahrefs DR, Backlinks), загружает их прошлые версии через CDX API (а также `waybackpy` / `wayback` в качестве failover-решений), очищает от мусора, определяет язык и классифицирует сайт по вхождениям ключевых слов.

Идеально подходит для поиска освобождающихся доменов (дропов) под тематики гемблинг, бизнес или игры.

---

## 🚀 Основной функционал

*   **Работа с Архивом (CDX API):** Получает до 200 снимков сайта с кодом 200 и типом `text/html`, распределенных по всей истории домена (до 5-10 снимков в год). Встроенный механизм обхода капчи/ошибок 429 от Web Archive (умные паузы и ротация User-Agent).
*   **Двойное резервирование (Failover):** Если CDX API недоступен, скрипт автоматически пытается получить данные через библиотеки `waybackpy` и `wayback`.
*   **Умная классификация:** Оценивает содержимое снимков на основе вхождений слов. Используются весовые коэффициенты HTML-зон (`title` - 3 балла, `h1` - 3 балла, `meta` - 2, `body` - 1). Доступные тематики: `Game`, `Business`, `Chess`, `News`, `Delivery`, `Bad`, `Other`.
*   **Строгая фильтрация контента:**
    *   Пропускает снимки с нецелевыми языками (по умолчанию ожидаются только `en` и `de`).
    *   Снимки с подозрительными китайско-японскими (азиатскими) кодировками помечаются как `Bad`.
    *   Сайт, который когда-либо менял тематику на спам или язык (Language Switch), отбрасывается полностью.
*   **Определение "Frozen" сайтов:** Программа сравнивает старейший и новейший снимки. Если их наполнение совпадает более чем на 95%, сайт помечается как `Frozen` (заброшенный).
*   **Исправленное распознавание редиректов:** Корректно обрабатывает Wayback Machine URL-обертки, различает кросс-доменные редиректы (блокируются) от валидных изменений (http→https, www↔non-www, изменения пути).
*   **Многопоточность:** Сайты обрабатываются параллельно (по умолчанию в 2 потока), внутри каждого домена снимки также обрабатываются параллельно (до 3 снимков одновременно).
*   **Оптимизация скорости:**
    *   **Resume support:** Автоматически продолжает работу после прерывания, пропуская уже обработанные домены
    *   **Early stop:** Останавливает анализ домена после получения 3 голосов за одну категорию
    *   **Кэширование CDX:** Избегает повторных запросов к CDX API для одного домена
    *   **Умный парсинг HTML:** Обрабатывает только первые 150KB страницы, пропускает полный парсинг body при наличии 20+ совпадений в title/h1/meta
    *   **Таймбаджет:** Ограничивает время обработки одного домена 120 секундами

## ⚙️ Требования к запуску

Для запуска скрипта вам не нужно устанавливать Python, если вы используете скомпилированный `.exe` файл (`analyzer.exe`).

Рядом с программой (или скриптом `analyzer.py`) должен находиться **Excel-файл** со списком доменов:

1.  **Название:** `sites_to_check.xlsx`
2.  **Структура столбцов:**
    *   `Domain` (или `Target`, `URL`, `site`) — сам домен (например, `example.com`).
    *   `Domain Rating` (или `DR`) — рейтинг домена по Ahrefs.
    *   `Backlinks / Followed` (или `Followed`) — количество Dofollow-ссылок.

> **Важно:** Перед началом парсинга программа отсечет все строки, не подходящие под лимит: **DR >= 7** и **Followed >= 10**. Вы можете изменить эти лимиты в исходном коде (`analyzer.py`) в блоке Configuration.

## 📘 Инструкция по использованию

1.  Разместите ваш исходный файл `sites_to_check.xlsx` в директории с запускным файлом `analyzer.exe`.
2.  Запустите программу двойным кликом по `.exe` файлу или через командную строку.
3.  Процесс анализа будет выводиться в консоль. Программа напишет, какие сайты она пропустила (с причиной пропуска), а какие отправила в итоговый список.
4.  **Resume support:** Если программа прервется (закрытие, ошибка, перезагрузка), просто запустите её снова — она автоматически продолжит с места остановки, используя кэш `results_cache.pkl`.
5.  После завершения парсинга в папке появится файл:
    **`final_results.xlsx`**
    
    В нем будут отражаться только подходящие вам сайты, а также дополнительная информация:
    *   Итоговая тематика
    *   Количество проанализированных и пропущенных снапшотов
    *   Определенный язык
    *   Статус `Frozen` (Yes/No)
    *   Success Level (1 = CDX API, 2 = waybackpy, 3 = wayback)

## 🔒 Настройка прокси (опционально)

Если web.archive.org заблокирован на вашем IP, вы можете использовать прокси-серверы.

### Как настроить прокси:

1.  Создайте файл `proxies.txt` в той же папке, где находится `analyzer.exe` (или `analyzer.py`)
2.  Добавьте прокси по одному на строку в любом из поддерживаемых форматов:
    ```
    http://1.2.3.4:8080
    http://user:password@1.2.3.4:8080
    socks5://1.2.3.4:1080
    socks5://user:password@1.2.3.4:1080
    ```
3.  Запустите программу как обычно — прокси будут использоваться автоматически

### Особенности работы с прокси:

*   **Автоматическая ротация:** Прокси чередуются по кругу (round-robin)
*   **Умное удаление:** Если прокси падает 3 раза подряд, он автоматически удаляется из ротации
*   **Graceful fallback:** Если все прокси исчерпаны, программа переключается на прямое подключение
*   **Опциональность:** Если файл `proxies.txt` отсутствует или пуст — программа работает без прокси (прямое подключение)

### Для SOCKS5 прокси:

Установите дополнительную зависимость:
```bash
pip install requests[socks]
```

Программа автоматически предупредит вас, если обнаружит SOCKS прокси в `proxies.txt`, но зависимость не установлена.

## 💻 Сборка из исходников (Для разработчиков)

Скрипт оптимизирован под сборку с помощью компилятора **Nuitka** в один исполняемый файл для удобной передачи пользователям. Функция `get_base_path()` гарантирует, что пути к файлам `.xlsx` не сломаются во временной папке Nuitka Onefile.

**Требования:**
*   Python 3.10+ (рекомендуется Python 3.13, т.к. 3.14 имеет экспериментальную поддержку в Nuitka)
*   Nuitka 4.0.5+

**Установка зависимостей:**
```bash
pip install pandas openpyxl bs4 lxml requests langdetect waybackpy wayback nuitka zstandard
```

**Для использования SOCKS5 прокси (опционально):**
```bash
pip install requests[socks]
```

**Команда для сборки EXE (рекомендуемая):**
```bash
nuitka --onefile --windows-console-mode=attach --enable-plugin=anti-bloat --jobs=8 --assume-yes-for-downloads --nofollow-import-to=tkinter --nofollow-import-to=matplotlib --nofollow-import-to=IPython --nofollow-import-to=pytest --nofollow-import-to=setuptools --include-package-data=openpyxl --include-package-data=lxml --include-package-data=bs4 --include-package-data=langdetect --follow-import-to=langdetect --output-filename=analyzer.exe analyzer.py
```

**Для систем с ограниченной памятью (без LTO):**
```bash
nuitka --onefile --windows-console-mode=attach --enable-plugin=anti-bloat --jobs=4 --assume-yes-for-downloads --include-package-data=openpyxl --include-package-data=lxml --include-package-data=bs4 --include-package-data=langdetect --follow-import-to=langdetect --output-filename=analyzer.exe analyzer.py
```

> **Примечание:** Установка `zstandard` обеспечивает сжатие onefile, уменьшая размер .exe на 30-40%. Параметр `--jobs` можно настроить под количество ядер вашего процессора.

## 🛠 Конфигурация / Кастомизация (`analyzer.py`)

Все основные настройки находятся вверху файла:

-   **`MIN_DOMAIN_RATING = 7`** (Минимальный DR для парсинга)
-   **`MIN_BL_FOLLOWED = 10`** (Минимальное количество follow ссылок)
-   **`SNAPSHOT_LIMIT = 200`** (Максимальное количество снапшотов на домен)
-   **`THREADS = 4`** (Кол-во потоков для доменов - параллельная обработка)
-   **`FETCH_THREADS = 2`** (Максимум одновременных HTTP запросов к web.archive.org на домен)
-   **`ANALYZE_THREADS = 6`** (CPU-bound потоки для парсинга HTML на домен)
-   **`API_SLEEP = 0.3`** (Пауза между CDX запросами в секундах)
-   **`REQ_TIMEOUT = 12`** (Таймаут для загрузки снапшотов в секундах)
-   **`CDX_TIMEOUT = 15`** (Таймаут для CDX API запросов в секундах)
-   **`DOMAIN_TIME_BUDGET = 60`** (Максимальное время обработки одного домена в секундах)
-   **`ALLOWED_LANGS = {"en", "de"}`** (Список разрешенных языков, остальное = мусор)
-   **`SESSION_POOL_SIZE = 3`** (Количество keep-alive сессий для ротации)
-   **`RATE_LIMIT_THRESHOLD = 3`** (Количество 429/503 ошибок для активации throttling)

При необходимости, списки ключевых слов для категорий находятся в переменных `GAME_KEYWORDS`, `BUSINESS_KEYWORDS`, `CHESS_KEYWORDS`, `DELIVERY_KEYWORDS`, `NEWS_KEYWORDS`, `BAD_KEYWORDS`.
