# Wayback Domain Analyzer

Утилита на Python для массового, автоматического анализа тематики доменов с использованием исторических снимков (snapshots) из сервиса Web Archive (Wayback Machine). 

Программа фильтрует домены по входным метрикам (Ahrefs DR, Backlinks), загружает их прошлые версии через CDX API (а также `waybackpy` / `wayback` в качестве failover-решений), очищает от мусора, определяет язык и классифицирует сайт по вхождениям ключевых слов.

Идеально подходит для поиска освобождающихся доменов (дропов) под тематики гемблинг, бизнес или игры.

---

## 🚀 Основной функционал

*   **Работа с Архивом (CDX API):** Получает до 40 последних снимков сайта с кодом 200 и типом `text/html`. Встроенный механизм обхода капчи/ошибок 429 от Web Archive (умные паузы и ротация User-Agent).
*   **Двойное резервирование (Failover):** Если CDX API недоступен, скрипт автоматически пытается получить данные через библиотеки `waybackpy` и `wayback`.
*   **Умная классификация:** Оценивает содержимое снимков на основе вхождений слов. Используются весовые коэффициенты HTML-зон (`title` - 3 балла, `h1` - 3 балла, `meta` - 2, `body` - 1). Доступные тематики: `Game`, `Business`, `Bad`, `Other`.
*   **Строгая фильтрация контента:**
    *   Пропускает снимки с нецелевыми языками (по умолчанию ожидаются только `en` и `de`).
    *   Снимки с подозрительными китайско-японскими (азиатскими) кодировками помечаются как `Bad`.
    *   Сайт, который когда-либо менял тематику на спам или язык (Language Switch), отбрасывается полностью.
*   **Определение "Frozen" сайтов:** Программа сравнивает старейший и новейший снимки. Если их наполнение совпадает более чем на 95%, сайт помечается как `Frozen` (заброшенный).
*   **Распознавание редиректов:** Отслеживает 301/302/Meta Refresh редиректы, чтобы анализировать конечный URL, и блокирует сайты, которые перенаправляют на чужие подозрительные домены.
*   **Многопоточность:** Сайты обрабатываются параллельно (по умолчанию в 2 потока — для минимизации блокировок от Архива).

## ⚙️ Требования к запуску

Для запуска скрипта вам не нужно устанавливать Python, если вы используете скомпилированный `.exe` файл (`WaybackAnalyzer.exe`).

Рядом с программой (или скриптом `analyzer.py`) должен находиться **Excel-файл** со списком доменов:

1.  **Название:** `sites_to_check.xlsx`
2.  **Структура столбцов:**
    *   `Domain` (или `Target`, `URL`, `site`) — сам домен (например, `example.com`).
    *   `Domain Rating` (или `DR`) — рейтинг домена по Ahrefs.
    *   `Backlinks / Followed` (или `Followed`) — количество Dofollow-ссылок.

> **Важно:** Перед началом парсинга программа отсечет все строки, не подходящие под лимит: **DR >= 7** и **Followed >= 10**. Вы можете изменить эти лимиты в исходном коде (`analyzer.py`) в блоке Configuration.

---

## 📘 Инструкция по использованию

1.  Разместите ваш исходный файл `sites_to_check.xlsx` в директории с запускным файлом `WaybackAnalyzer.exe`.
2.  Запустите программу двойным кликом по `.exe` файлу.
3.  Процесс анализа будет выводиться в консоль. Программа напишет, какие сайты она пропустила (с причиной пропуска), а какие отправила в итоговый список.
4.  После завершения парсинга в папке появится файл:
    **`final_results.xlsx`**
    
    В нем будут отражаться только подходящие вам сайты (`Game` или `Business`), а также дополнительная информация:
    *   Итоговая тематика
    *   Количество проанализированных и пропущенных снапшотов
    *   Определенный язык
    *   Статус `Frozen` (Yes/No)

## 💻 Сборка из исходников (Для разработчиков)

Скрипт оптимизирован под сборку с помощью компилятора **Nuitka** в один исполняемый файл для удобной передачи пользователям. Функция `get_base_path()` гарантирует, что пути к файлам `.xlsx` не сломаются во временной папке Nuitka Onefile.

**Требования:**
*   Python 3.10+
*   Компилятор C (MSVC или MinGW) — требуется для Nuitka на Windows.

**Установка зависимостей:**
```bash
pip install pandas bs4 requests langdetect waybackpy wayback nuitka
```

**Команда для сборки EXE:**
```bash
python -m nuitka --standalone --onefile --jobs=1 --include-package-data=langdetect --include-package=openpyxl --windows-console-mode=force --output-filename=WaybackAnalyzer analyzer.py
```
> Примечание: Убран `fake_useragent`. В последнем обновлении добавлен встроенный список User-Agents.

## 🛠 Конфигурация / Кастомизация (`analyzer.py`)

Все основные настройки находятся вверху файла:

-   **`MIN_DOMAIN_RATING = 7`** (Минимальный DR для парсинга)
-   **`MIN_BL_FOLLOWED = 10`** (Минимальное количество follow ссылок)
-   **`SNAPSHOT_LIMIT = 40`** (Сколько снапшотов вытягивать через API Web Archive)
-   **`THREADS = 2`** (Кол-во потоков. Больше делать не рекомендуется, Web Archive быстро даст бан 429)
-   **`ALLOWED_LANGS = {"en", "de"}`** (Список разрешенных языков, остальное = мусор)

При необходимости, списки ключевых слов для категорий находятся в переменных `GAME_KEYWORDS`, `BUSINESS_KEYWORDS`, `BAD_KEYWORDS`.
